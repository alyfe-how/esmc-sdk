# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 4.1.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**DO NOT** create a public GitHub issue for security vulnerabilities.

Instead, please email us at: **security@esmc-sdk.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### What to Expect

- **Acknowledgment:** Within 24 hours
- **Initial Assessment:** Within 72 hours
- **Status Updates:** Every 7 days until resolved
- **Resolution Timeline:** Depends on severity

### Severity Levels

| Level | Response Time | Example |
|-------|--------------|---------|
| Critical | 24 hours | Remote code execution |
| High | 72 hours | Authentication bypass |
| Medium | 1 week | Information disclosure |
| Low | 2 weeks | Minor issues |

### Recognition

We appreciate security researchers who:
- Report vulnerabilities responsibly
- Allow reasonable time for fixes
- Don't exploit vulnerabilities

Responsible reporters will be:
- Credited in release notes (if desired)
- Thanked on our security acknowledgments page

## Security Measures

ESMC SDK includes:

- **Obfuscated Code:** Core components are protected
- **License Validation:** Server-side authentication
- **No Secrets in Repo:** API keys require user setup
- **Machine ID Binding:** Licenses tied to devices
